# テンプレート・キャッチアップ（空白期間明けの最初の一手）

このテンプレートの鮮度は還流ループ（使われるたびに直って正本へ戻る）で保たれるが、
ループは**使われた時にしか回らない**。空白期間中の世界のドリフト——Playwright/Chromium・
Node・Claude Code・ブラウザポリシー——はここで一括検出し、**最初の実装に入る前に**返済する。
立ち上げ日がテンプレート修理で不意打ちされるのを、予定された一手目に変えるための工程。

## いつ実行するか

- テンプレートから新しいゲームを立ち上げた**最初のセッション**（最初の実装より前）
- `framework/`・`.claude/` の最終コミットから **3ヶ月以上**空いた状態でのセッション開始時
  （`git log -1 --format=%cs -- framework/ .claude/` で判定）
- どちらでもなければ実行不要（毎セッションやるものではない）

## 手順

### 1. 配線の実証（まず動かす）

```bash
(cd framework && npm i) && (cd harness && npm i)
node harness/run.js
```

- スターター（または現行ゲーム）の全プロファイルが green なら、Playwright・Chromium・Node の
  基本配線は生きている。**green ならドリフトの大半は「無し」で確定**——以降は差分確認だけ。
- 落ちたら、それこそが検出したかったドリフト。エラーから原因（API 変更・ブラウザ挙動の変更）を
  特定して直す。直したら**同じセッションで `/framework-backflow`**（次の利用者への返済）。

### 2. 依存・実行環境のドリフト

- `npm outdated`（framework/ と harness/ の両方）。Playwright のメジャーが上がっていたら
  changelog の breaking changes を確認してから上げ、上げたら手順1を再実行する
- Node の現行 LTS と実行環境のバージョンの乖離を確認する

### 3. Claude Code 側のドリフト

- `.claude/agents/*.md` の frontmatter を点検する（`model:` の指名が現行ラインナップに
  存在するか。advisor の上位モデル指定は現行の上位モデル名に追随させる）
- スキル・エージェントが正しくロードされているか（このスキルが起動したこと自体が一部の証明）
- Claude Code の新機能でこのワークフローの手作業を置き換えられるものがないか一考する
  （必須ではない。採用するなら分岐判断として `DECISIONS.md` に残す）

### 4. 見直しの適用と還流

- 検出・修正したドリフトのうち、再発しうる欠陥クラスは `harness/KNOWLEDGE.md` のバグ台帳へ
- `framework/`・`.claude/` を直した場合は **`/framework-backflow` で正本へ**還流する。
  これが「空白期間の負債は最初の利用者が返し、正本で全員が回収する」の仕組みの本体——
  ここを飛ばすと修理が写しに閉じて、次のゲームがまた同じ負債から始まる
- 何も無ければ何も書かない（無風の記録は不要。ハーネスの green が記録代わり）
