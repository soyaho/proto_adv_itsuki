# PR監視：1時間ごとの能動チェックイン（send_later ループ）はデフォルト off

- 種別: ユーザーの流儀
- 日付: 2026-07-15
- 出自: proto_ae（PR #7 の監視ループに対して「そもそもデフォルトで off にしたい」との明示指示）

## 事実

PR を作成・購読したあとに send_later で「変化がなくても1時間ごとに PR 状態を再確認して
再アームする」自己チェックインのループを組むのは、**ユーザーの明示依頼が無い限りやらない**。

## 運用

- `subscribe_pr_activity`（webhook 購読）だけで待つ。レビューコメント・CI失敗はイベントで
  届き、セッションが起こされる
- ただし webhook には届かないものがある（**マージ・CI成功・新規プッシュ・コンフリクト化**）。
  これらは次にユーザーが話しかけたとき、または明示依頼されたときに確認すれば足りる
- ユーザーが「監視して」「babysit して」等と明示した場合のみ、能動チェックインを組んでよい
  （その場合も頻度と止め時を伝えること）

## 経緯

システム側の既定手順（PR購読後に send_later で1時間ごとの自己チェックインを推奨）に従って
proto_ae でループを組んだところ、ユーザーの意向は「既定 off」だった。権限クラシファイアも
「ユーザー承認の無い自己永続タスク」として再アームを拒否しており、判断は一致している。
複数プロジェクト（chiiku・proto_seiichi）でも同じループを組んでいた形跡があったため、
テンプレート正本に還流して次作以降の既定とする。
