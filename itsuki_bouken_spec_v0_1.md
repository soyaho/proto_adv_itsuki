## 0. 感情仕様（この一文が全設計判断の最終基準）

> **「考えて、やってみたら、できた」**

- プレイヤーは6歳（いつきくん本人がモデル。プレイヤーも本人）
- 実装判断に迷ったら、この一文に照らして「考える余地を残すか」「やってみた実感が出るか」「できた瞬間が本人の手柄に見えるか」で決めること

## 1. コンセプト

いつきくんの実際の朝のルーティン2つ（朝ごはんの準備／幼稚園に行く準備）を、横長ワンシーンのタップ操作ゲームとして再現する。遊ぶことが現実の朝のリハーサルになる。

## 2. 不可侵の設計原則

1. **✗（バツ・不正解表示・減点・スコア）を一切出さない。** スコア、星、タイマー、失敗演出は存在しない
2. **タップ＝いつきくんへの指示。** タップした場所へいつきくんキャラが歩いて移動し、触れて、初めて結果が起きる。指と結果の間に必ず本人の体が挟まる
3. **何をタップしても世界が反応する。** 解決に関係ないオブジェクトも必ず小さなリアクションを返す（揺れる、音が鳴る、など）。世界の反応そのものがヒント
4. **困るのは世界ではなく、いつきくん本人。** 届かない・足りない・順番違いのときは、いつきくんが「うーん」と首をかしげるアニメで表現する。これが唯一の「不成立」フィードバック
5. **ゴール条件を文字で表示しない。** 完成形の絵がゴールを語る。〔反証反映・所見1〕完成形の絵は**世界に常設する**：シーン1は各席のランチョンマットに置き場所のうすい形（ゴースト）を最初から描く。シーン2はカバンの中にうすい持ち物の形を描く。クリア時の絵（全員でいただきます／カバンを背負って出発）はその完成
6. **文字は最小限。使う場合はひらがなのみ**（対象年齢6歳、未就学）

## 3. 共通仕様

### 3.1 画面・操作
- 横長ワンシーン固定（スクロールなし）。想定デバイス: iPhone / iPad 横持ち
- タップ可能オブジェクトの当たり判定は最小 60×60px
- いつきくんの移動: タップ位置へ歩行アニメで移動（トコトコ感。移動速度は約 250px/秒 目安。可愛さ優先で調整可）
- 移動中に別の場所をタップした場合は目的地を上書き

### 3.2 キャラクター表現
- いつきくん: シンプルな図形ベースで可。歩行・ジャンプ・「うーん」（首かしげ）・喜び（両手上げ）の4モーション必須
- 顔にリアルな造形は不要。丸＋点目＋口程度で愛嬌を出す
- パパ・ママ: 静的でよいが、状況に応じたセリフ吹き出し（ひらがな短文 or 絵文字的アイコン）を出せること

### 3.3 インタラクションの型
- **持つ/置く**: オブジェクトをタップ→いつきくんが取りに行く→頭上に持つ→置き先をタップ→運んで置く
- **注ぐ/入れる**: 容器を持った状態で対象をタップすると注ぐ・入れる
- **塗る**: 対象を複数回タップで進行、見た目の変化で完了を示す
- 〔反証反映・所見3〕**持っている間の文法**: 持っている間も世界の反応は変わらない（原則3）。**置けるのは置き先（席・テーブル・カバン等の受け側）だけ**。取った場所をもう一度タップすると**戻せる**（キャンセル）。それ以外をタップした場合、いつきくんはそこへ歩いて対象の小さなリアクションを起こすだけで、持ち物は持ったまま。持ち物で手が塞がって詰む状態は存在しない
- 〔反証反映・所見5〕**あける（捨てる）**: 水の入った容器を持ってシンクをタップすると、水をシンクにあける
- 〔反証反映・所見7〕**ふく**: 床の水たまりをタップすると、いつきくんが拭く（きゅっきゅっ＋ぴかっ。楽しい系の音）
- 〔反証反映・所見6〕**まとめ持ち**: おしぼりの山・コップの重ねは束のまま持てる。置き先をタップするたびに1つずつ置く。**箸は1膳ずつ**（「3人ぶん」を数える考えるポイントを保つ）
- ドラッグは使わない。すべてタップの連鎖で完結させる（6歳の精密操作負荷を避ける）

### 3.4 フィードバック
- 〔反証反映・所見4〕**受理は即時・結果は接触時**: タップの瞬間、タップ地点に小さな受理マーク（波紋）を出し、いつきくんが即座に歩き出す。オブジェクトのリアクションと結果は、いつきくんが触れた瞬間に起きる（原則2の「体が挟まる」を守りつつ、操作の受理は即時に帰属させる）
- 成立: 短い効果音＋オブジェクトの定着アニメ
- 不成立: いつきくんの「うーん」モーション＋関連ヒントの微アニメ（例: 氷が入らない→水面がタプタプ揺れる）
- シーンクリア: いつきくんの喜びモーション＋完成形の絵が1〜2秒静止して見える演出
- 効果音: Web Audio API で合成（外部音源ファイル不使用）。iOS WebKit 対策として初回タップで `ctx.resume()` するアンロックパターンを必ず実装

### 3.5 世界のルール（1本の理）〔反証反映・所見2・7〕

> **「ふたのない みずは、はこぶと こぼれる」**

- 開いた容器（コップ・フタの開いた水筒）に水が入った状態で歩くと、少し歩いたところでこぼれる。水は床の水たまりになる（拭ける）。容器は空になるだけで、他の進行は巻き戻さない——罰ではなく物理
- フタのある閉じた容器（みずさし・フタを閉めた水筒）は運んでも安全
- このルールはシーン1の水運びで最初に体験され、シーン2の「水筒のフタを閉めてからカバンへ」の理由になる。場所ごとに別の理屈を発明しない

## 4. シーン1: あさごはんのじゅんび

### 4.1 舞台
ダイニング。テーブル（席3つ: パパ席・ママ席・いつき席。各席にランチョンマット＝ゴール絵）、食器棚（お箸・コップの重ね）、おしぼり置き場、シンク（蛇口）＋みずさし、炊飯器。パパとママはシーン開始時テーブル付近に立っている。シーン開始時にママが一言吹き出し（「じゅんび おねがいね」）。

### 4.2 タスク（順不同で達成可能）
| タスク | 手順 | 考えるポイント |
|---|---|---|
| お箸を出す | 食器棚→箸を1膳取る→席に置く ×3膳 | **数**: 家族は3人。1膳だけだと箸のない席のパパ/ママが箸のない手元を見て待つ絵になる |
| おしぼりを出す | おしぼりの山をまとめて持つ→各席をタップして1つずつ置く | — |
| コップに水 | コップの重ねを席に置く（3席）→みずさし（フタつき）を持つ→蛇口をタップで水を入れる→席のコップをタップして注ぐ ×3 | **順番**: コップを持ったまま蛇口で水を入れると、運ぶ途中でこぼれる（3.5の世界ルール）。「フタのあるみずさしなら運べる」に気づく。こぼれた水は拭くとまた楽しい、程度の軽さで |

### 4.3 クリア条件
3席すべてに箸・おしぼり・水入りコップが揃う → 3人が席に着き「いただきます」（手を合わせるアニメ＋効果音）。

## 5. シーン2: ようちえんのじゅんび

### 5.1 舞台
キッチン〜玄関。冷蔵庫（上: 冷凍室に氷／下段）、蛇口＋シンク、水筒（フタは水筒のとなり）、お弁当箱、お箸セット（弁当箱の陰に少し隠す）、カバン（中に持ち物のうすい形＝ゴール絵）、日焼け止めチューブ、鏡、玄関ドア。ママがキッチンにいる。

### 5.2 タスク
| タスク | 手順 | 考えるポイント |
|---|---|---|
| 水筒の用意 | 冷凍庫を開ける→氷を水筒に入れる→水筒を持って蛇口で水を注ぐ→フタを水筒に閉める→カバンへ | **順番**: 先に水を満たすと氷が入らない（水面タプタプ→「うーん」）。水はシンクにあけてやり直せる（3.3）。フタを閉めずに水入り水筒を運ぶとこぼれる（3.5）——フタの意味が体でわかる |
| お弁当を入れる | お弁当箱→カバンへ、お箸セット→カバンへ | **抜け漏れ**: お箸セットは弁当箱の陰に少し隠して配置 |
| 日焼け止め | チューブを持つ→鏡をタップ→鏡の前に立つと鏡が顔を大きく映す→**鏡の中の顔**をタップで塗る（3ヶ所）→全部塗るとテカッと光って完了 | **状態を見る**: 塗り残し（頬だけ白い等）が鏡の中で見た目でわかる。〔反証反映・所見8〕タップ対象はあくまで世界のオブジェクト（鏡）——「タップ＝いつきくんへの指示」の文法を破らない |
| 忘れ物ゲート | カバンが完成していない状態で玄関ドアをタップすると、ママが足りないものを一言吹き出しで教える（「おはしは？」等）＋そのオブジェクトが小さく揺れる | **抜け漏れ**: 忘れて出発しようとして初めて気づく（ゲート時のみのヒント） |

### 5.3 クリア条件
カバンに水筒・弁当・箸が入り、日焼け止め完了 → カバンをタップして背負う→玄関ドアへ→「いってきます！」で手を振ってクリア。

## 6. 日替わり変化（プロトでは各シーン2バリエーション）

シーン開始時にランダムで1つ選択。将来的に実日付シードで日替わり化する前提の構造にしておく（選択関数を1ヶ所に分離）。

- シーン1-A: 通常
- シーン1-B: **パパがもう席に着いている**。〔反証反映・所見10〕パパ席には箸・おしぼり・水がすべて揃い済み（パパが自分で用意した）。残り2席分だけでクリア。3膳目を持っていくとパパが「もうあるよ」→食器棚に戻せる（3.3のキャンセル）
- シーン2-A: 通常
- シーン2-B: **冷凍室に氷がない**（開けると空）。ママをタップすると「れいぞうこの したよ」と教えてくれる。冷蔵庫の下段を開けると氷がある（ママに聞かず自力で見つけてもよい）

## 7. 画面フロー

```
タイトル「いつきくんの ぼうけん」
  └ タップでスタート（ここで AudioContext アンロック）
      └ シーン選択（あさごはん / ようちえん の2枚絵ボタン）
          └ シーンプレイ → クリア演出 → シーン選択に戻る
```

- セーブ不要。リロードで最初から
- 〔反証反映・所見9〕もどるボタンは上部バー（遊び場の外）に置き、押すとバー内が「やめる？ うん／まだ」の2択に変わる**2段階確認**にする（誤タップ1回で進行が消えない）

## 8. 技術仕様

- 単一 `index.html`（CSS/JS インライン）。外部依存・CDN・ビルド工程なし
- 描画: Canvas（テンプレートのハーネス基盤 `tapTargets()`・論理座標系に合わせる。SVG推奨からの変更は DECISIONS.md D10）
- アニメ: Canvas 内で自前補間（固定タイムステップ）。テスト用に `?ts=N` で加速可能にする（KNOWLEDGE 原則39）
- 音: Web Audio API 合成音のみ。BGMなし、効果音のみ
- localStorage / sessionStorage 不使用
- iPhone Safari 横持ちで動作確認できること（`viewport` 設定、`touch-action` 制御、ダブルタップズーム抑止）
- テスト注入（実入力主義の例外）: `?ts=N`（タイムスケール）・`?v1=A|B`・`?v2=A|B`（バリエーション固定）

## 9. スコープ外（v0.1では作らない）

- スコア・実績・報酬システム（原則として永久に作らない可能性が高い）
- 音声読み上げ
- 3シーン目以降
- キャラクターカスタマイズ
- 実日付連動の日替わり（構造だけ準備）
- 〔反証反映・所見10〕リプレイを引っ張る仕掛け（周回変化の拡充）——プロトの目的は「1周の手応え」の検証

## 10. 受け入れ基準

1. 6歳が説明なしで「タップ→いつきくんが動く」を理解できる（チュートリアル文字なしで成立）
2. 全タップ可能オブジェクトが何らかの反応を返す
3. ✗表現・スコア・タイマーが存在しない
4. 箸の数／水の順番／氷の順番の3つの「考えるポイント」が、文字説明なしに絵と反応だけで気づける
5. クリア時、直前の自分の操作が結果につながったことが画面上で見える
6. iPhone実機（Working Copy経由）でタップ遅延・ズーム誤動作なく遊べる

## 11. 実装後のセルフレビュー観点（Claude Code向け）

- 感情仕様の一文に照らして、「考える」が「当てる」に劣化していないか（正解表示・誘導矢印を足していないか）
- いつきくんの移動時間が思考の間として機能しているか（速すぎ＝連打ゲー化に注意）
- 「うーん」モーションが罰に見えないか（音を暗くしない、画面を赤くしない）
- 〔反証反映〕ランチョンマット・カバンのゴースト表示が「考える」を奪っていないか（プレイテスト判定基準へ）
